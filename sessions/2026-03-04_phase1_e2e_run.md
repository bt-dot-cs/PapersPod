## End-to-End Run Debugging

Multiple pipeline runs needed to clear API/config issues. All bugs were in config/credentials, not code.

### Bug 1 — arXiv date filter: 0 papers returned
**Root cause**: `max_results=query.max_papers` (=3) fetched only the 3 most recent papers sorted by `submittedDate` descending. For a topic with 106k papers, all 3 were from March 2026, past the `2026-01-01` end date.

**Attempted fix 1**: `submittedDate:[20220101* TO 20260101*]` in query string → arXiv API returns HTTP 500 (wildcards not supported).

**Attempted fix 2**: `submittedDate:[20220101 TO 20260101]` → API accepts it but ignores it (server-side date filtering not functional in the Lucene endpoint).

**Final fix**: Changed sort to `SortCriterion.Relevance` with `max_results=max(query.max_papers * 20, 50)`. Relevance sort returns a mix of dates; Python-side date filter handles the rest. Updated `test_build_search_query_no_disciplines` test from exact equality check to substring check.

### Bug 2 — Script agent: `host='Alex'` fails Pydantic validation
**Root cause**: Claude returned `{"host": "Alex", ...}` using the character name from the prompt description rather than the identifier `"A"`. The prompt says `HOST A (Alex):` and Claude latched onto "Alex".

**Fix**: Added `_HOST_ALIASES` dict and `_normalize_host()` in `script_agent.py` to map `"alex"→"A"`, `"jordan"→"B"`, `"host a"→"A"`, `"host b"→"B"` before constructing `DialogueTurn`.

### Bug 3 — ElevenLabs voice IDs: placeholder values in .env
`.env` had `your_host_a_voice_id_here` and `your_host_b_voice_id_here` as literal values (the IDs were written in comments). Fixed by user editing `.env`.

### Bug 4 — ElevenLabs 401: API key missing `text_to_speech` permission
ElevenLabs API key was created without TTS scope. User regenerated key with correct permissions.

### Bug 5 — ElevenLabs 404: voice ID `yFUu5jJAN31dt6KeE1fk2` not found
Leading `y` was accidentally prepended to the voice ID. Fixed by user correcting `.env`.

### Bug 6 — ElevenLabs 402: free plan cannot use library voices via API
Ann (Malaysian Chinese) and Elio (Spanish accent) are community library voices; free tier cannot use these via API.

**Fix**: Switched to default voices (available on free tier). `.env` now has two blocks — free voices active, premium commented out for easy swap when upgrading:
