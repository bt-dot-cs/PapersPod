## Key Implementation Decisions Made

### `core/config.py` — Import-time validation
Config raises `ValueError` at import time if required env vars are missing. `tests/conftest.py` sets fake values at module level (before any test imports happen) using `os.environ.setdefault()`. This pattern allows real keys in shell env to take precedence.

### `core/arxiv_client.py` — Design notes
- `_build_search_query()` combines topic + discipline category codes (`cs.LG`, `cs.CL`, etc.)
- Empty disciplines list → topic-only query (no category filter)
- Study period extraction calls Claude only when `study_data_period` is set in `QueryParameters`
- Rate limiting via `asyncio.sleep(ARXIV_RATE_LIMIT_SECONDS)` between results
- `arxiv.Client.results()` is mocked in tests; no real API calls

### `core/semantic_scholar_client.py` — Design notes
- Uses `SemanticScholar.get_paper(f"ARXIV:{arxiv_id}")` for lookup
- TLDR handled as either dict (`{"text": ...}`) or object with `.text` attribute (S2 SDK returns either)
- `None` result → warning + paper returned unchanged
- Exception handling catches all S2 failures gracefully
- `_compute_citation_velocity()` floored at 1 year denominator to prevent division by zero on new papers
